## Not changing the new `maxFeePercent` to the old `maxFeePercent` can save a gas
If we change the old `maxFeePercent` with the same value it doesnt throw an error, 
since the logic of `if` statement will only revert if the sum of all fee percent are greater than the new `maxFeePercent`.

```
if (
            feesEconomics.protocolFeePercent +
                feesEconomics.subjectFeePercent +
                feesEconomics.referralFeePercent +
                feesEconomics.holdersFeePercent >
            maxFeePercent_
        ) revert InvalidFeeDefinition();
        feesEconomics.maxFeePercent = maxFeePercent_;
```
And if we try to change the `maxFeePercent` with the same value it will not change anything and waste the gas for call that function.
So it is better to change the operator `>` into `>=`, so we cant change it to same value. 
By changing the operator it does saves the gas. let see the gas report below
